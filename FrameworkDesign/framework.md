# DeepReproduction 项目框架设计

## 项目目的
DeepReproduction 是一个基于 LLM Agents 的漏洞复现框架，旨在自动化完成漏洞的复现与验证过程。通过智能体协作，该框架能够从漏洞信息采集、环境构建、POC 生成到最终验证的全流程自动化，提高漏洞复现的效率和准确性。

## 项目架构

### 核心组件
1. **阶段模块 (Stages)**
   - `knowledge`：知识采集阶段，负责聚合漏洞资料并生成结构化的知识模型
   - `build`：环境构建阶段，负责生成 Dockerfile、构建脚本并组织漏洞版本的编译过程
   - `poc`：POC 生成与执行阶段，负责基于漏洞知识和构建产物生成最小复现代码与运行脚本
   - `verify`：验证阶段，负责综合前后版本运行日志判断漏洞是否被稳定触发与修复

2. **状态管理**
   - `AppState`：定义 LangGraph 运行时共享状态，作为各阶段节点之间传递信息的统一容器

3. **数据模型**
   - `TaskModel`：任务输入结构，承载单个 CVE 任务的基础信息、仓库信息和参考链接
   - `KnowledgeModel`：漏洞知识结构，作为知识阶段输出及其他阶段的统一输入
   - `BuildArtifact`：构建产物结构，包含构建过程的结果和状态
   - `PoCArtifact`：POC 产物结构，包含 POC 代码和执行结果
   - `VerifyResult`：验证结果结构，包含验证结论和原因

4. **工具 (Tools)**
   - 提供各种辅助功能，如 Docker 操作、文件操作、Git 操作、日志处理等

## 工作流程

### 整体流程
1. **任务加载**：从 YAML 文件加载任务信息，创建 `TaskModel`
2. **初始化状态**：设置初始状态，包括任务信息、工作空间路径、重试计数等
3. **知识采集**：调用知识阶段，生成 `KnowledgeModel`
4. **环境构建**：调用构建阶段生成构建产物，执行构建过程
5. **POC 生成与执行**：调用 POC 阶段生成 POC 代码和运行脚本，执行 POC 并捕获结果
6. **验证**：调用验证阶段分析前后版本运行日志，判断漏洞是否被稳定触发与修复
7. **结果输出**：返回最终验证结果和任务状态

### 流程控制
- 使用 LangGraph 构建状态图，定义各阶段之间的流转关系
- 构建阶段和 POC 阶段支持重试机制
- 验证阶段根据验证结果决定最终状态

## 简化原则

1. 保留四阶段主流程，不改变项目的核心职责边界
2. 合并原先过薄的 `agents/` 与 `nodes/`，减少文件跳转成本
3. 删除未接入或重复的占位模块，降低概念数量
4. 统一通过文件头注释说明职责，提升目录可读性
